# 쓰레드캡쳐 기획서

## 1. 프로젝트 개요

### 제품 정의
`Thread 글 요약&분석기`는 사용자가 직접 저장한 Threads 캡처 파일을 한 폴더에 모아두면, 프로그램이 이를 읽어 Gemini로 요약·분석하고 카테고리별 결과 PDF로 정리해주는 개인용 배치 프로그램이다.

### 현재 방향
- PC 전용 로컬 앱
- 입력은 사용자가 직접 저장한 `jpg / png / pdf`
- Threads 내부 자동 조작 없음
- 브라우저 자동화 없음
- Google 계정 제어 없음
- Gemini API 키는 앱 화면에서 세션용으로 입력

## 2. 해결하려는 문제
- Threads에서 저장한 좋은 글을 나중에 다시 읽지 않게 된다.
- 캡처 파일이 쌓여도 분류와 재열람이 어렵다.
- 외국어 글이나 맥락이 복잡한 글은 다시 이해하는 데 시간이 든다.
- 개인 아카이브 관점에서 PDF 정리본이 필요하다.

## 3. 핵심 목표
- 원본 폴더에 파일을 넣고 한 번에 처리할 수 있을 것
- AI가 `요약 + 상세 해설 + 핵심 포인트 + 인사이트 + 카테고리`를 생성할 것
- 결과 PDF를 카테고리 폴더에 자동 저장할 것
- 이미 처리한 파일은 다음 실행에서 자동 스킵할 것
- 새 다운로드 파일이 들어오면 자동 실행할 수 있을 것

## 4. 핵심 원칙
- Threads 계정에 로그인해서 자동 클릭, 자동 스크롤, 자동 수집하지 않는다.
- 프로그램은 로컬 폴더의 파일만 읽고 처리한다.
- Gemini API 키는 세션용 입력값으로만 사용한다.
- 결과물과 메타데이터는 로컬에 저장한다.

## 5. 사용자 흐름

### 기본 흐름
1. 사용자가 크롬에서 GoFullPage 등으로 Threads 글을 캡처한다.
2. 캡처 파일이 `source-files` 폴더로 저장된다.
3. 프로그램에서 원본 폴더와 결과 폴더를 확인한다.
4. 카테고리를 입력하거나 수정한다.
5. Gemini 모델을 선택한다.
6. `시작` 버튼을 누르거나 `Auto Run`을 켠다.
7. 프로그램이 파일을 순차 처리한다.
8. 결과 PDF가 카테고리별 폴더에 저장된다.

### 자동 실행 흐름
1. 사용자가 `Auto Run` 토글을 켠다.
2. 프로그램이 원본 폴더를 주기적으로 스캔한다.
3. 새 파일이 감지되면 자동으로 배치 실행을 시작한다.
4. 이미 처리된 파일은 스킵한다.

## 6. 입력 / 출력 구조

### 입력
- `jpg`
- `png`
- `pdf`

### 출력
- 카테고리별 결과 PDF
- Markdown 리포트
- JSON 메타데이터

### 폴더 구조
```text
data/
  source-files/
    *.jpg
    *.png
    *.pdf
  archive/
    경제/
    바이브코딩/
    기타/
    meta/
```

## 7. Gemini 모델 방향
- UI의 `Gemini 2.0 Free`는 내부적으로 `gemini-2.5-flash-lite`
- UI의 `Gemini 3.0 Free Preview`는 내부적으로 `gemini-3.1-flash-lite-preview`
