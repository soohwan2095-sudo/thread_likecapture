# 쓰레드캡쳐 화면구성서

## 1. 문서 목적
현재 구현된 단일 화면 UI 구성을 정리한다.

## 2. 전체 구조
이 앱은 사이드바 기반 다중 화면 구조가 아니라, 한 화면에서 설정과 실행과 결과 확인을 모두 처리하는 작업형 UI다.

```text
상단 Hero
좌측 제어 패널
우측 상태 / 파일 / 결과 패널
```

## 3. 상단 Hero 영역

### 구성 요소
- 프로그램 이름: `Thread 글 요약&분석기`
- 짧은 설명 문구
- 현재 선택된 Gemini 모델 표시

### 목적
- 앱 정체성 전달
- 현재 모델 상태 즉시 확인

## 4. 좌측 제어 패널

### 4.1 원본 폴더 카드
- 원본 폴더 경로 입력
- 결과 저장 경로 입력
- `폴더 열기` 버튼
- `경로 적용` 버튼

### 4.2 AI 종류 카드
- `Gemini 2.0 Free`
- `Gemini 3.0 Free Preview`
- 선택된 모델 강조 표시

### 4.3 카테고리 생성 카드
- 카테고리 전체값 입력 textarea
- 카테고리 태그 미리보기
- `생성 버튼`

### 4.4 보안 설정 카드
- `Gemini API Key` 입력칸
- 세션 전용 사용 안내
- 브라우저 자동화 없음 안내

### 4.5 Auto Run 카드
- 자동 실행 토글 스위치
- 폴링 주기 입력
- 새 파일 감지 시 자동 실행 안내

### 4.6 Batch Run 카드
- 한 번에 처리한다는 설명 문구
- `시작` 버튼

## 5. 우측 패널

### 5.1 현재 상태 카드
- 상태 메시지 표시
- `파일 새로고침` 버튼
- 신규 / 스킵 / 실패 요약 카드

### 5.2 원본 폴더 파일 목록 카드
- 현재 원본 폴더의 지원 파일 목록
- 파일명
- 상태
- 카테고리
- `원본 열기`
- `결과 열기`

### 5.3 최근 결과 카드
- 최근 처리된 결과 리스트
- 제목
- 카테고리
- 한줄 요약

### 5.4 선택한 결과 카드
- 선택한 작업의 상세 결과
- 카테고리
- 한줄 요약
- 상세 해설
- 핵심 포인트
- 인사이트
- `원본`
- `결과 PDF`

## 6. 시각 방향

### 현재 스타일
- 밝은 배경 기반
- 베이지 / 오렌지 / 블루그레이 조합
- 둥근 카드형 레이아웃
- 대시보드보다 작업 도구에 가까운 느낌

### 폰트
- UI는 번들된 `malgun.ttf` 사용
- 한글 표시가 깨지지 않도록 고정

## 7. 사용자 흐름
1. 원본 폴더와 결과 폴더 확인
2. Gemini 모델 선택
3. 카테고리 입력
4. API 키 입력 또는 데모 모드 사용
5. `시작` 또는 `Auto Run`
6. 상태 카드에서 진행 확인
7. 최근 결과에서 상세 결과 열람

## 8. 현재 구현 반영 사항
- 다중 화면 구조 제거
- 단일 작업 화면으로 단순화
- 자동 실행 토글 추가
- 결과 상세 패널 유지
- 최근 결과 리스트 유지
- 한글 폰트 적용 및 표시 문제 수정 완료
