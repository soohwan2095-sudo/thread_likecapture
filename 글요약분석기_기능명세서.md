# 글 요약 분석기 기능명세서

## 1. 문서 목적

현재 구현된 `글 요약 분석기`의 기능 기준을 정리한다.

## 2. 제품 범위

- 로컬 데스크톱 앱
- 원본 폴더 기반 배치 처리
- 원본 파일 목록 기반 선택 처리
- Gemini 분석
- 카테고리별 결과 PDF 생성
- 결과 PDF에 원본 파일 병합
- Markdown / JSON 메타데이터 저장

## 3. 기능 목록

1. 원본 폴더 경로 설정
2. 결과 폴더 경로 설정
3. Gemini 모델 선택
4. 카테고리 입력 및 폴더 생성
5. 세션용 Gemini API 키 입력
6. 전체 배치 실행
7. 체크박스 기반 선택 파일 분석
8. 새 파일 감지 자동 실행
9. 원본 파일 목록 조회
10. 최근 결과 목록 조회
11. 결과 상세 보기
12. 결과 PDF / Markdown / JSON 저장
13. 결과 PDF에 원본 PDF 또는 이미지 병합
14. 내용 기반 한글 12자 이내 결과 파일명 생성
15. 이미 결과 PDF가 있는 파일 건너뛰기

## 4. 처리 기준

- 지원 입력 형식은 `pdf`, `png`, `jpg`, `jpeg`이다.
- 전체 `시작` 버튼은 원본 폴더의 새 파일 전체를 처리한다.
- `선택 분석` 버튼은 체크박스로 선택한 새 파일만 처리한다.
- 체크박스는 아직 처리되지 않은 `new` 상태 파일에만 활성화된다.
- API 키가 비어 있으면 데모 모드로 처리한다.
- API 키가 있으면 Gemini 분석 요청을 보낸다.
- 스킵 기준은 실제 결과 PDF 파일 존재 여부이다.
- 기존 DB에 기록만 있고 결과 PDF가 없으면 다시 처리할 수 있다.

## 5. 결과 파일명 규칙

- 결과 PDF 파일명은 분석 결과의 `title`, `oneLineSummary`, 원본 파일명 순서로 후보를 만든다.
- 후보 문자열에서 한글만 추출한다.
- 파일명 본문은 12자 이내로 제한한다.
- 같은 이름의 PDF, Markdown, JSON이 이미 있으면 12자 제한 안에서 숫자를 붙인다.
- 한글 후보가 없으면 `요약결과`를 기본 이름으로 사용한다.

예시:

```text
금리인하전망.pdf
금리인하전망2.pdf
요약결과.pdf
```

## 6. 결과 PDF 구성

결과 PDF는 다음 순서로 구성한다.

1. Gemini 요약/분석 페이지
2. 원본 파일

원본 파일 처리 방식:

- 원본이 PDF이면 기존 PDF 페이지를 뒤에 그대로 복사한다.
- 원본이 PNG/JPG/JPEG이면 이미지 페이지를 새로 만들어 뒤에 붙인다.

## 7. 분석 결과 스키마

```text
title
category
oneLineSummary
detailedExplanation
keyPoints[]
insights[]
uncertaintyNotes[]
```

## 8. 상태값

파일 상태:

```text
new
processed
failed
processing
```

작업 상태:

```text
pending
analyzing
completed
failed
```

## 9. 데이터 승계

앱 이름 변경 전 DB 파일명은 `thread_likecapture.db`였다.

새 DB 파일명은 `content_summary_analyzer.db`이다. 앱 실행 시 새 DB가 아직 없고 예전 DB가 있으면 예전 DB를 새 DB 파일명으로 자동 변경한다. 이 로직은 기존 처리 기록과 최근 결과를 잃지 않기 위한 호환 처리이다.

## 10. 보안 기준

- 브라우저 자동화를 하지 않는다.
- 구글 계정 로그인을 자동 제어하지 않는다.
- 쿠키를 수집하지 않는다.
- API 키를 영구 저장하지 않는다.
- Gemini 요청에는 사용자가 지정한 원본 파일만 포함한다.
