# 글 요약 분석기 기획서

## 1. 프로젝트 개요

`글 요약 분석기`는 사용자가 직접 저장한 PDF, PNG, JPG 파일을 원본 폴더에 모아두면 Gemini로 요약과 분석을 수행하고, 카테고리별 결과 PDF로 정리해주는 개인용 로컬 배치 프로그램이다.

이 앱은 특정 서비스 전용 캡처 도구가 아니라, 사용자가 저장한 다양한 글 캡처와 문서 파일을 관리하는 범용 요약/분석 도구로 운영한다.

## 2. 기본 방향

- 로컬 데스크톱 앱으로 동작한다.
- 입력은 사용자가 직접 저장한 `pdf`, `png`, `jpg`, `jpeg` 파일이다.
- 특정 서비스 계정 로그인이나 브라우저 자동 조작을 하지 않는다.
- Gemini API 키는 화면에서 세션 전용으로 입력한다.
- 결과물과 메타데이터는 로컬 `data/archive` 아래에 저장한다.
- 기존 프로젝트명 `Thread LikeCapture`는 `글 요약 분석기` / `content-summary-analyzer`로 변경한다.

## 3. 핵심 목표

- 원본 폴더에 파일을 넣고 한 번에 처리할 수 있다.
- 원본 폴더 안에서 원하는 개별 파일만 선택해 분석할 수 있다.
- AI가 요약, 상세 해설, 핵심 포인트, 인사이트, 카테고리를 생성한다.
- 결과 PDF 파일명은 내용을 짐작할 수 있는 한글 12자 이내 이름으로 저장한다.
- 결과 PDF 앞부분에는 요약/분석 내용을 넣고, 뒷부분에는 원본 파일을 함께 붙인다.
- 이미 결과 PDF가 생성된 파일은 다음 실행에서 건너뛴다.
- 새 파일이 들어오면 자동 실행 옵션으로 처리할 수 있다.

## 4. 사용자 흐름

1. 사용자가 분석할 글 캡처 또는 문서 파일을 `source-files` 폴더에 저장한다.
2. 앱에서 원본 폴더와 결과 폴더를 확인한다.
3. 카테고리를 입력하거나 수정한다.
4. Gemini 모델을 선택한다.
5. 필요하면 Gemini API 키를 입력한다.
6. 전체 처리하려면 `시작` 버튼을 누른다.
7. 일부 파일만 처리하려면 원본 파일 목록에서 체크박스로 파일을 고른 뒤 `선택 분석`을 누른다.
8. 결과 PDF가 카테고리별 폴더에 저장된다.
9. 결과 PDF에서 AI 요약/분석 내용과 원본 파일을 함께 확인한다.

## 5. 현재 구현 범위

- 단일 작업 화면 UI
- 원본 폴더 / 결과 폴더 지정
- 카테고리 직접 입력 및 폴더 생성
- Gemini 모델 선택
- 세션용 Gemini API 키 입력
- 전체 배치 실행
- 원본 파일 목록 체크박스 기반 선택 분석
- 새 파일 감지 후 자동 실행
- 최근 결과 목록과 상세 보기
- 결과 PDF / Markdown / JSON 메타데이터 생성
- 결과 PDF 뒤에 원본 PDF 또는 이미지 병합
- 결과 파일명을 한글 12자 이내 내용 기반 이름으로 저장
- 실제 결과 PDF가 존재하는 파일만 건너뛰기
- 기존 `thread_likecapture.db`를 `content_summary_analyzer.db`로 자동 승계

## 6. 이번 변경 요약

- 제품명을 `Thread LikeCapture`에서 `글 요약 분석기`로 변경했다.
- 패키지명과 실행 파일명을 `content-summary-analyzer`로 변경했다.
- Threads 전용 표현을 제거하고 PDF/이미지 기반 범용 분석 도구로 정리했다.
- 파일 목록에 체크박스를 추가해 개별 파일 선택 분석을 지원했다.
- 결과 PDF 파일명 생성 규칙을 내용 기반 한글 12자 이내로 변경했다.
- 결과 PDF에 원본 파일을 뒤에 병합하도록 변경했다.
- 배치 스킵 기준을 DB 기록 존재가 아니라 실제 결과 PDF 존재 기준으로 개선했다.
