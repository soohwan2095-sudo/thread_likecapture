# UI 간소화 개선 요구사항

작성일: 2026-04-14  
대상 프로젝트: `content-summary-analyzer` / 글 요약 분석기  
목적: 현재 설정 카드가 많은 UI를 단순화하고, 실제 사용에 필요한 핵심 동작만 남기는 개선 작업 기준을 정리한다.

## 1. 현재 문제 인식

현재 UI는 한 화면에 다음 요소가 한꺼번에 노출되어 있다.

- 원본 폴더 설정
- 결과 폴더 설정
- AI 모델 선택
- 카테고리 생성
- Gemini API key 입력
- 자동 실행 설정
- 전체 배치 실행
- 원본 파일 목록
- 최근 결과
- 선택 결과 상세

기능은 많지만 사용자가 실제로 자주 쓰는 흐름에 비해 화면이 복잡하다. 특히 “설정 카드”가 많아서 처음 쓰는 입장에서는 무엇부터 눌러야 하는지 판단하기 어렵다.

따라서 개선 방향은 기능 확장이 아니라 기능 축소와 흐름 단순화다.

## 2. 새 UI의 핵심 방향

새 UI는 다음 4가지 작업만 명확하게 보이면 된다.

1. 개별 파일 하나를 선택해서 변환
2. 폴더 안에 들어간 파일 전체 변환
3. Gemini API key 입력
4. 이미 변환된 파일의 DB 이력을 삭제해서 다시 변환 가능하게 만들기

이 외의 설정성 기능은 기본값으로 숨기거나, 필요할 때만 열리는 고급 설정으로 빼는 것이 좋다.

## 3. 목표 화면 구조

추천하는 단일 화면 구조:

```text
┌──────────────────────────────────────────────┐
│ 글 요약 분석기                                │
│ 원본 폴더: D:\...\data\source-files   [열기] │
│ API Key: [************************]           │
├──────────────────────────────────────────────┤
│ [폴더 전체 변환] [새로고침]                   │
├──────────────────────────────────────────────┤
│ 파일 목록                                     │
│ □ 파일명              상태       작업         │
│ □ sample.pdf          미변환     [변환]       │
│ □ done.pdf            변환완료   [결과열기] [이력삭제] │
│ □ failed.png          실패       [다시변환]   │
└──────────────────────────────────────────────┘
```

핵심은 사용자가 화면을 열자마자 다음 질문에 답을 얻는 것이다.

- 어떤 파일이 들어와 있는가?
- 어떤 파일이 아직 변환 전인가?
- 어떤 파일이 이미 변환됐는가?
- 지금 하나만 변환할 수 있는가?
- 전체를 한 번에 변환할 수 있는가?
- 변환 이력을 삭제하고 다시 돌릴 수 있는가?

## 4. 유지할 기능

반드시 남길 기능:

- Gemini API key 입력
- 원본 폴더 파일 목록 표시
- 파일별 상태 표시
  - `미변환`
  - `변환 중`
  - `변환 완료`
  - `실패`
- 개별 파일 변환 버튼
- 폴더 전체 변환 버튼
- 결과 PDF 열기 버튼
- 원본 파일 열기 버튼
- 변환 이력 삭제 버튼
- 실패 파일 다시 변환
- 새로고침

## 5. 숨기거나 제거할 기능

기본 화면에서 숨길 기능:

- AI 모델 카드
- 카테고리 생성 카드
- 자동 실행 카드
- 경로 적용 버튼 중심의 설정 UI
- 최근 결과 카드
- 상세 결과 카드
- 히어로 영역
- 긴 설명 문구

완전히 제거하지 않고 고급 설정으로 이동해도 되는 기능:

- Gemini 모델 선택
- 카테고리 목록 수정
- 결과 저장 폴더 변경
- 감시 주기 설정
- 자동 실행 토글

기본값으로 처리 가능한 항목:

- 원본 폴더: 현재 DB 설정 또는 `data/source-files`
- 결과 폴더: 현재 DB 설정 또는 `data/archive`
- 카테고리: 현재 기본값 `경제, 바이브코딩, 기타`
- 모델: 현재 기본값 `gemini-2.5-flash-lite`

## 6. 개별 파일 변환 요구사항

파일 목록의 각 파일 행에는 `변환` 버튼이 있어야 한다.

동작 기준:

- 상태가 `미변환`이면 `변환` 버튼 노출
- 상태가 `실패`이면 `다시변환` 버튼 노출
- 상태가 `변환 완료`이면 `결과열기`, `이력삭제` 버튼 노출
- 상태가 `변환 중`이면 버튼 비활성화

현재 백엔드 `run_batch`는 `selected_file_paths`를 이미 지원한다. 따라서 개별 파일 변환은 선택 파일 경로 1개만 전달하는 방식으로 구현할 수 있다.

예상 호출 흐름:

```text
runBatch(sourceFolder, categories, geminiModel, apiKey, [file.path])
```

## 7. 폴더 전체 변환 요구사항

상단 또는 파일 목록 위에 `폴더 전체 변환` 버튼을 둔다.

동작 기준:

- 원본 폴더 안의 지원 파일 전체를 대상으로 실행한다.
- 이미 결과 PDF가 존재하는 파일은 기존 로직처럼 자동 스킵한다.
- 실패 파일은 재처리 대상으로 포함할지 정책을 정해야 한다.

추천 정책:

- `미변환` 파일은 처리
- `실패` 파일도 다시 처리
- `변환 완료` 파일은 스킵

현재 백엔드 스킵 기준은 “DB에 `output_pdf_path`가 있고 해당 PDF 파일이 실제 존재하는가”이므로, 실패 파일은 결과 PDF가 없으면 다시 처리될 수 있다.

## 8. API Key 입력 요구사항

API key 입력은 화면에서 가장 잘 보이는 위치에 둔다.

권장 UI:

```text
Gemini API Key [____________________]
```

표시 정책:

- API key는 저장하지 않는다.
- 입력값은 현재 실행 세션에서만 사용한다.
- API key가 비어 있으면 데모 모드로 실행된다는 사실을 짧게 표시한다.

권장 문구:

```text
비워두면 데모 결과로 변환됩니다. 실제 분석에는 Gemini API key가 필요합니다.
```

긴 보안 설명은 기본 화면에서는 빼고, 필요하면 도움말 또는 고급 설정에 넣는다.

## 9. 변환 이력 삭제 요구사항

사용자가 요청한 핵심 신규 기능이다.

목적:

- 한 번 변환된 파일은 현재 로직상 다시 변환되지 않는다.
- 하지만 사용자가 결과가 마음에 들지 않거나 API key 없이 데모로 돌린 경우 다시 실제 분석하고 싶을 수 있다.
- 따라서 DB에서 해당 파일의 job 이력을 삭제해 다시 `미변환` 상태로 돌아가게 해야 한다.

권장 버튼명:

```text
이력삭제
```

또는 더 명확하게:

```text
다시 변환 가능하게
```

권장 동작:

1. 사용자가 `이력삭제` 클릭
2. 확인 메시지 표시
3. DB의 해당 job 삭제
4. 필요하면 기존 결과 PDF/Markdown/JSON 파일 삭제 여부 선택
5. 파일 목록 새로고침
6. 해당 원본 파일 상태가 `미변환`으로 표시됨

## 10. 이력 삭제 시 산출물 처리 정책

구현 전 결정해야 할 중요한 정책이다.

추천 기본 정책:

- DB 이력은 삭제한다.
- 기존 결과 PDF/Markdown/JSON 파일은 기본적으로 삭제하지 않는다.
- 대신 확인 창에서 “결과 파일도 함께 삭제” 옵션을 제공한다.

이유:

- 사용자가 실수로 이력 삭제를 눌러도 기존 결과 파일이 사라지지 않는다.
- 다시 변환하면 새 결과 파일이 생성되고, 파일명 중복은 기존 저장 로직이 suffix로 처리할 수 있다.

단순 구현 우선 버전:

- 1차 구현에서는 DB 이력만 삭제한다.
- 산출물 삭제 옵션은 2차 개선으로 둔다.

## 11. 필요한 백엔드 변경

새 Tauri command가 필요하다.

추천 command:

```rust
delete_job_history_by_source_path(source_path: String) -> Result<(), String>
```

또는 job id 기준:

```rust
delete_job_history(job_id: String) -> Result<(), String>
```

권장 방식:

- UI에서 `SourceFileItem.jobId`가 있으면 job id 기준으로 삭제한다.
- job id가 없고 경로만 있으면 source path 기준으로 삭제할 수 있게 보조 command를 둔다.

`src-tauri/src/storage/mod.rs`에 추가할 DB 함수:

```rust
pub fn delete_job_history(&self, job_id: &str) -> Result<(), AppError>
```

예상 SQL:

```sql
DELETE FROM jobs WHERE id = ?1
```

주의:

- 삭제 전에 job detail을 읽어두면 산출물 경로를 나중에 삭제 옵션에 활용할 수 있다.
- 1차 구현은 DB row 삭제만 수행해도 된다.

## 12. 필요한 프론트엔드 변경

주요 변경 대상:

- `app/src/app/App.tsx`
- `app/src/lib/api.ts`
- `app/src/types/models.ts`
- `src-tauri/src/commands/mod.rs`
- `src-tauri/src/storage/mod.rs`
- `src-tauri/src/main.rs`

프론트엔드 변경 방향:

- 히어로 영역 제거
- 설정 카드 대부분 제거 또는 접기
- 파일 목록 중심 UI로 재구성
- 각 파일 행에 작업 버튼 추가
- `handleStartSingleFile(file.path)` 추가
- `handleDeleteHistory(file)` 추가
- 전체 변환 버튼은 기존 `handleStartBatch()` 활용
- 선택 체크박스 기반 다중 선택 기능은 제거하거나 숨김

예상 함수:

```ts
async function handleStartSingleFile(filePath: string) {
  await handleStartBatch(false, [filePath]);
}

async function handleDeleteHistory(file: SourceFileItem) {
  if (!file.jobId) return;
  await deleteJobHistory(file.jobId);
  await refreshSourceFiles();
  await refreshRecentJobs();
}
```

## 13. UI 문구 초안

상단:

```text
글 요약 분석기
원본 폴더에 들어온 PDF/이미지를 Gemini로 요약하고 결과 PDF로 저장합니다.
```

API key:

```text
Gemini API Key
비워두면 데모 결과로 변환됩니다.
```

버튼:

```text
폴더 전체 변환
새로고침
원본 폴더 열기
결과 폴더 열기
변환
다시변환
결과열기
원본열기
이력삭제
```

상태:

```text
미변환
변환 중
변환 완료
실패
건너뜀
```

이력 삭제 확인:

```text
이 파일의 변환 이력을 삭제할까요?
삭제 후 다시 변환할 수 있습니다. 기존 결과 파일은 삭제하지 않습니다.
```

## 14. 구현 우선순위

1. 새 이력 삭제 command 추가
2. 파일 목록 행에 `변환`, `결과열기`, `이력삭제` 버튼 추가
3. 화면에서 히어로/설정 카드/최근 결과/상세 결과를 제거하거나 숨김
4. API key 입력과 폴더 전체 변환 버튼만 상단에 배치
5. 개별 파일 변환 함수 추가
6. 변환 후 목록 자동 새로고침
7. 실패 메시지를 파일 행에 간단히 표시
8. 고급 설정 접기 영역 추가

## 15. 이번 개선에서 하지 않아도 되는 것

이번 단순화 작업에서는 다음은 하지 않아도 된다.

- 대규모 디자인 개편
- 대시보드 통계
- 상세 결과 미리보기
- 자동 실행 고도화
- 카테고리 관리 화면
- 검색/필터 고도화
- 다중 선택 작업
- 파일 hash 기반 중복 감지
- PDF 내부 미리보기

이 기능들은 나중에 필요해졌을 때 추가해도 된다. 지금 목표는 사용자가 헷갈리지 않고 “API key 넣고, 하나 변환하거나, 폴더 전체 변환하고, 필요하면 이력 삭제 후 다시 변환”하는 것이다.

## 16. 최종 목표 문장

새 UI는 다음 문장으로 설명될 수 있어야 한다.

> 원본 폴더에 파일을 넣고, API key를 입력한 뒤, 파일 하나를 변환하거나 폴더 전체를 변환한다. 이미 변환된 파일은 자동으로 건너뛰며, 다시 변환하고 싶으면 이력삭제를 눌러 미변환 상태로 되돌린다.
