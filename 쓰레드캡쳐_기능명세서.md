# 쓰레드캡쳐 기능명세서

## 1. 문서 목적
이 문서는 현재 구현된 `Thread 글 요약&분석기`의 기능 기준을 정리한다.

## 2. 제품 범위
- 로컬 데스크톱 앱
- 원본 폴더 기반 배치 처리
- Gemini 분석
- 카테고리별 결과 PDF 생성

## 3. 기능 목록
1. 원본 폴더 경로 설정
2. 결과 폴더 경로 설정
3. Gemini 모델 선택
4. 카테고리 입력 및 폴더 생성
5. 세션용 Gemini API 키 입력
6. 수동 배치 실행
7. 새 파일 감지 자동 실행
8. 원본 파일 목록 조회
9. 최근 결과 목록 조회
10. 결과 상세 보기
11. PDF / Markdown 저장
12. 이미 처리한 파일 스킵

## 4. 상세 요구사항

### 4.1 원본 폴더 설정
- 사용자는 원본 파일 폴더 경로를 입력할 수 있어야 한다.
- 프로그램은 해당 폴더 안의 `jpg`, `png`, `pdf` 파일을 스캔해야 한다.
- `폴더 열기` 버튼으로 탐색기를 열 수 있어야 한다.

### 4.2 결과 폴더 설정
- 사용자는 결과 저장 루트 폴더를 입력할 수 있어야 한다.
- 프로그램은 카테고리별 하위 폴더와 `meta` 폴더를 생성할 수 있어야 한다.

### 4.3 Gemini 모델 선택
- 사용자는 두 가지 모델 프리셋 중 하나를 선택할 수 있어야 한다.
- 표시명과 실제 모델명은 아래와 같다.

```text
Gemini 2.0 Free -> gemini-2.5-flash-lite
Gemini 3.0 Free Preview -> gemini-3.1-flash-lite-preview
```

### 4.4 카테고리 생성
- 사용자는 카테고리를 쉼표 또는 줄바꿈으로 입력할 수 있어야 한다.
- 프로그램은 입력한 카테고리별 폴더를 생성할 수 있어야 한다.
- 결과 파일은 분류된 카테고리 폴더에 저장되어야 한다.

### 4.5 API 키 입력
- 사용자는 Gemini API 키를 화면에서 입력할 수 있어야 한다.
- API 키는 현재 실행 세션에서만 사용해야 한다.
- API 키를 앱 설정에 영구 저장하지 않아야 한다.

### 4.6 수동 배치 실행
- 사용자는 `시작` 버튼으로 배치를 수동 실행할 수 있어야 한다.
- 프로그램은 원본 폴더의 파일을 순차 처리해야 한다.
- 처리 결과는 `신규 처리`, `스킵`, `실패` 건수로 요약되어야 한다.

### 4.7 자동 실행
- 사용자는 `Auto Run` 토글로 자동 실행을 켜고 끌 수 있어야 한다.
- 사용자는 폴링 주기(ms)를 설정할 수 있어야 한다.
- 자동 실행이 켜져 있으면 새 파일 감지 시 배치를 시작해야 한다.
- 이미 처리한 파일은 자동 실행에서도 스킵해야 한다.

### 4.8 파일 목록 조회
- 프로그램은 원본 폴더의 파일 목록을 표시해야 한다.
- 각 항목에 아래 정보가 보여야 한다.
  - 파일명
  - 상태
  - 분류 카테고리
  - 원본 열기 버튼
  - 결과 열기 버튼

### 4.9 최근 결과 조회
- 프로그램은 최근 처리 결과를 카드 형태로 보여야 한다.
- 각 카드에는 아래 정보가 포함되어야 한다.
  - 제목
  - 카테고리
  - 한줄 요약

### 4.10 결과 상세 보기
- 사용자는 최근 결과 카드를 눌러 상세 내용을 볼 수 있어야 한다.
- 상세 보기에는 아래 내용이 포함되어야 한다.
  - 카테고리
  - 한줄 요약
  - 상세 해설
  - 핵심 포인트
  - 인사이트
  - 원본 열기 버튼
  - 결과 PDF 열기 버튼

### 4.11 PDF / Markdown 생성
- 프로그램은 분석 결과로 Markdown을 생성해야 한다.
- 프로그램은 분석 결과로 PDF를 생성해야 한다.
- PDF에는 한글이 정상 출력되어야 한다.
- PDF는 카테고리 폴더에 저장되어야 한다.
- Markdown과 JSON 메타는 `archive/meta` 아래 저장되어야 한다.

### 4.12 중복 처리 방지
- 프로그램은 이미 처리 이력이 있는 파일을 다음 실행에서 스킵해야 한다.
- 스킵 기준은 기존 작업 레코드 기준으로 판단한다.

## 5. 분석 결과 스키마
```text
title
category
oneLineSummary
detailedExplanation
keyPoints[]
insights[]
uncertaintyNotes[]
```

## 6. 상태값

### 파일 상태
- `new`
- `processed`
- `failed`
- `processing`

### 작업 상태
- `pending`
- `analyzing`
- `completed`
- `failed`

- Threads 자동 클릭 없음
- Threads 자동 스크롤 없음
- Google 계정 로그인 제어 없음
- 쿠키 수집 없음
- API 키 영구 저장 없음
